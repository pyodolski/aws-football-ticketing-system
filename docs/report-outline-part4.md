# 📊 보고서 목차 (4장부터) - 실험 및 결과 분석

## 4. 부하 테스트 설계 및 준비 (Load Testing Design and Preparation)

### 4.1 부하 테스트 목적 및 범위 (Testing Objectives and Scope)

#### 4.1.1 테스트 목표 정의

- 오토스케일링 동작 검증
- 로드밸런서 트래픽 분산 확인
- 시스템 안정성 및 성능 측정
- 병목 지점 식별

#### 4.1.2 테스트 시나리오 설계

- **시나리오 1**: 로그인 페이지 - 대량 회원가입/로그인
- **시나리오 2**: 메인 페이지 - 경기 조회 및 예매 내역
- **시나리오 3**: 티켓 예매 - 좌석 조회 및 예매 시도

#### 4.1.3 성공 기준 정의 (Success Criteria)

- 인스턴스 자동 확장: 2개 → 5개 (목표 시간: 5분 이내)
- 인스턴스 자동 축소: 5개 → 2개 (목표 시간: 20분 이내)
- 응답 시간: 평균 < 500ms
- 에러율: < 5%

### 4.2 테스트 환경 구성 (Test Environment Setup)

#### 4.2.1 초기 시스템 상태

- 인스턴스 개수: 2개 (최소값)
- CPU 사용률: < 5%
- 메모리 사용률: ~40%
- 데이터베이스 상태: 정상 운영 중

#### 4.2.2 오토스케일링 설정 확인

- 최소 인스턴스: 2개
- 최대 인스턴스: 5개
- 목표 CPU 사용률: 10%
- 초기화 기간: 30초
- 안정화 기간: 30초

#### 4.2.3 모니터링 도구 설정

- GCP Console 모니터링
- 웹 기반 실시간 모니터링 위젯
- 로드밸런서 통계
- Cloud SQL 모니터링

### 4.3 부하 생성 메커니즘 (Load Generation Mechanism)

#### 4.3.1 부하 생성 방식

- 클라이언트 사이드 JavaScript 기반
- 초당 500명의 가상 사용자 시뮬레이션
- 실제 API 호출 (회원가입, 로그인, 조회)

#### 4.3.2 트래픽 패턴

- 회원가입: 50% (DB INSERT)
- 로그인: 50% (DB SELECT + JWT)
- 경기 조회: 60% (DB SELECT)
- 예매 내역: 40% (DB JOIN)
